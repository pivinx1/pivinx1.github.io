Building Apps for Windows Mobile

Go back to the main page

Building stuff for this platform sure is an interesting task, mainly because knowledge for it is nowadays quite scarce.

Probably the first thing to do would be to obtain some tools.

You will need:

  • Any version of Visual Studio between 2005 and 2010 (Visual Studio 2008 worked best for me), must be the Professional version. You can get it from the Internet Archive here.
  • An SDK (Software Development Kit) for your version of Windows Mobile of choice. Do note that versions targeted towards older versions of Windows Mobile should work on newer versions. You can get one from the Internet Archive here.
  • The .NET Compact Framework 2.0 installed on your development machine

You can omit any item on the list below as needed.

Additionally:

  • A Windows Mobile device of choice (mine was the HP iPAQ 214)
  • ActiveSync (on Windows XP and older) or Windows Mobile Device Center (on Windows Vista and 7) for syncing your program onto your device of choice

You don't need to have a Windows Mobile device on hand.

Visual Studio 2008 comes with the Device Emulator, which can emulate a Windows Mobile device as needed.

Your SDK should come with the ROM images for the Device Emulator by default.

The Code

As a matter of fact, writing code for the Windows Mobile platform at it's base is not much different from writing code for the Win32 platform.

Windows Mobile is based on Windows CE, but it supports both the Win32 API but in addition also supports the WinCE API.

Because of this, using the Win32 API for writing your app will allow an user to run your app on a Windows PC as well.

However, do note that some file-related things may become inherently broken due to fundamental differences in the Windows filesystem.
